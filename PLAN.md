# headapi — Implementation Plan

## Overview

`headapi` is a standalone Go service that implements the Tailscale REST API v2
(`api.tailscale.com/api/v2/…`) as a thin proxy/translation layer over the
headscale gRPC API. Its goal is to make headscale compatible with Tailscale
ecosystem tooling — primarily the Tailscale Kubernetes operator, the Tailscale
Terraform provider, and any other client that targets the Tailscale control-plane
API — without modifying headscale itself.

```
┌────────────────────────────┐        ┌─────────────────────────┐
│  Tailscale k8s operator /  │  REST  │         headapi          │  gRPC  │  headscale  │
│  Terraform provider / etc. │ ──────▶│  (this project)         │ ──────▶│             │
└────────────────────────────┘        └─────────────────────────┘        └─────────────┘
```

---

## Goals & Non-Goals

### Goals
- Implement the Tailscale API v2 surface that is practically mappable to headscale gRPC.
- Implement an OAuth 2.0 Client Credentials token endpoint (`POST /oauth/token`) so that the k8s operator's OAuth flow works.
- Be deployable as a single static binary or container, sidecar to headscale.
- Full TDD: every handler is driven by a table-driven `httptest`-based test before implementation.
- Clearly flag endpoints that have **no headscale equivalent** and return `501 Not Implemented` with a structured JSON body.

### Non-Goals
- Replacing or forking headscale.
- Implementing the Tailscale control-plane / coordination API (that is headscale's job).
- Perfect behavioural parity with tailscale.com for every edge case.

## Tech Stack

| Concern | Choice | Rationale |
|---|---|---|
| HTTP router | [`go-chi/chi`](https://github.com/go-chi/chi) v5 | lightweight, idiomatic, path params match OpenAPI `{param}` style |
| gRPC client | `google.golang.org/grpc` + generated headscale proto stubs | official headscale Go client |
| Config | [`kelseyhightower/envconfig`](https://github.com/kelseyhightower/envconfig) | simple, no magic |
| OAuth token store | in-memory `sync.Map` with HMAC-signed opaque tokens | avoids DB dependency; tokens are short-lived |
| Testing | stdlib `net/http/httptest` + `testify/assert` + `testify/mock` | no external test framework |
| Mocks | [`vektra/mockery`](https://github.com/vektra/mockery) v2 on `HeadscaleClient` interface | regenerated by `make generate` |
| Structured logging | `log/slog` (stdlib, Go 1.21+) | zero deps |
| Integration test | `ory/dockertest` + `hsic`/`haic` (mirrors headscale's own test framework) | true end-to-end confidence |

## Phase 1 — Authentication & OAuth Token Endpoint (Day 1–2)

This is the **critical enabler** for the Kubernetes operator. The operator uses
OAuth 2.0 Client Credentials (`grant_type=client_credentials`) to obtain a
short-lived bearer token, which it then uses to call the Tailscale API.

### Endpoint
```
POST /oauth/token          (NOT under /api/v2 — matches tailscale.com path)
```

### Design
- Accept `client_id` + `client_secret` as either HTTP Basic Auth or form body.
- `client_id` and `client_secret` are configured statically in headapi config
  (operators create them; headapi validates them against config or a simple DB table).
- On success, issue an opaque token: `HMAC-SHA256(secret, random_nonce + expiry)`,
  stored in `sync.Map[token → {scopes, expiry}]`.
- Response matches Tailscale's `{"access_token":"…","token_type":"Bearer","expires_in":3600}`.
- All `/api/v2/…` endpoints validate incoming bearer token against the in-memory store
  OR accept a headscale API key directly (for non-OAuth tooling like Terraform).

### Files
- `internal/api/auth.go` — `OAuthTokenHandler`, `BearerAuthMiddleware`
- `internal/api/auth_test.go` (written first)

### Tests (written before handler code)
```go
// auth_test.go — table-driven

    name       string
```

## Phase 2 — Devices (Day 2–3)

Maps Tailscale "devices" to headscale "nodes".

### Headscale gRPC → Tailscale REST mapping

| Tailscale REST | headscale gRPC |
|---|---|
| `GET /tailnet/{tailnet}/devices` | `ListNodes` (filter by user/tailnet) |
| `GET /device/{deviceId}` | `GetNode` |
| `DELETE /device/{deviceId}` | `DeleteNode` |
| `POST /device/{deviceId}/authorized` | `AuthApprove` / `AuthReject` |
| `POST /device/{deviceId}/expire` | `ExpireNode` |
| `POST /device/{deviceId}/name` | `RenameNode` |
| `POST /device/{deviceId}/tags` | `SetTags` |
| `GET /device/{deviceId}/routes` | `GetNode` (routes field) |
| `POST /device/{deviceId}/routes` | `SetApprovedRoutes` |
| `POST /device/{deviceId}/ip` | ⚠️ No equivalent — `501` |
| `POST /device/{deviceId}/key` | ⚠️ No equivalent — `501` |
| `PATCH /tailnet/{tailnet}/device-attributes` | ⚠️ No equivalent — `501` |
| `GET /device/{deviceId}/attributes` | ⚠️ No equivalent — `501` |
| `POST /device/{deviceId}/attributes/{key}` | ⚠️ No equivalent — `501` |
| `DELETE /device/{deviceId}/attributes/{key}` | ⚠️ No equivalent — `501` |

### Model translation (`internal/translate/device.go`)
```go
// NodeToDevice converts a headscale proto Node to a Tailscale API Device.
func NodeToDevice(n *headscalev1.Node) model.Device { … }
```

Key field mappings:
- `Node.Id` (uint64) → `Device.ID` (string, zero-padded to match Tailscale convention)
- `Node.Name` → `Device.Hostname`
- `Node.IpAddresses` → `Device.Addresses`
- `Node.User.Name` → `Device.User`
- `Node.LastSeen` → `Device.LastSeen` (RFC3339)
- `Node.Expiry` → `Device.KeyExpiryDisabled` (expiry.IsZero())
- `Node.ForcedTags` + `Node.ValidTags` → `Device.Tags`
- `Node.ApprovedRoutes` + `Node.AdvertisedRoutes` → `DeviceRoutes`

### Files
- `internal/model/device.go`
- `internal/translate/device.go`
- `internal/api/devices.go`
- `internal/api/devices_test.go` (written first — uses mock HeadscaleClient)

### Tests
Table-driven httptest tests per endpoint. Mock `HeadscaleClient.ListNodes` returns
a synthetic proto `[]*Node`. Assert JSON response body fields and status codes.
Test error paths: node not found → `404`, gRPC `PermissionDenied` → `403`.

## Phase 3 — Auth Keys (Day 3–4)

Maps Tailscale "auth keys" to headscale "pre-auth keys".

### Mapping

| Tailscale REST | headscale gRPC |
|---|---|
| `GET /tailnet/{tailnet}/keys` | `ListPreAuthKeys` |
| `POST /tailnet/{tailnet}/keys` | `CreatePreAuthKey` |
| `GET /tailnet/{tailnet}/keys/{keyId}` | `ListPreAuthKeys` + filter by ID |
| `DELETE /tailnet/{tailnet}/keys/{keyId}` | `ExpirePreAuthKey` then `DeletePreAuthKey` |
| `PUT /tailnet/{tailnet}/keys/{keyId}` | ⚠️ No headscale equivalent (key rotation) → `501` |

### Model translation (`internal/translate/key.go`)
```go
func PreAuthKeyToKey(k *headscalev1.PreAuthKey) model.Key { … }
func KeyRequestToCreatePreAuthKey(r model.CreateKeyRequest, user string) *headscalev1.CreatePreAuthKeyRequest { … }
```

Key field mappings:
- `PreAuthKey.Id` → `Key.ID`
- `PreAuthKey.Key` → `Key.Key` (only returned on creation)
- `PreAuthKey.Reusable` → `Key.Capabilities.Devices.Create.Reusable`
- `PreAuthKey.Ephemeral` → `Key.Capabilities.Devices.Create.Ephemeral`
- `PreAuthKey.Expiration` → `Key.Expires`
- `PreAuthKey.AclTags` → `Key.Capabilities.Devices.Create.Tags`

### Note on `{keyId}` — headscale doesn't support lookup by ID
`ListPreAuthKeys` returns all keys for a user. headapi must iterate and match.
The `{tailnet}` path param is used as the headscale username for pre-auth key
scoping (or resolved via config).

## Phase 5 — Policy File / ACL (Day 4–5)

### Mapping

| Tailscale REST | headscale gRPC |
|---|---|
| `GET /tailnet/{tailnet}/acl` | `GetPolicy` |
| `POST /tailnet/{tailnet}/acl` | `SetPolicy` |
| `POST /tailnet/{tailnet}/acl/preview` | ⚠️ No headscale equivalent → `501` |
| `POST /tailnet/{tailnet}/acl/validate` | ⚠️ No headscale equivalent → `501` |

The Tailscale API supports both JSON and HuJSON policy formats (negotiated via
`Content-Type` and `Accept` headers). headscale's `SetPolicy` accepts HuJSON.
headapi should pass the body through verbatim when `Content-Type: application/hujson`,
and re-serialize when `Content-Type: application/json`.

make the upgrade path clear — replace `NotImplemented()` with a real gRPC call.

## Phase 7 — Stub Endpoints (Day 5–6)

The following groups have **no headscale gRPC equivalent** and will return `501`
with a JSON body `{"message":"not implemented by headapi"}`:

- **Webhooks** — `GET/POST/PATCH/DELETE /tailnet/{tailnet}/webhooks`, etc.
- **Logging** — all `/tailnet/{tailnet}/logging/…` endpoints.
- **Contacts** — `GET/PATCH /tailnet/{tailnet}/contacts/…`.
- **UserInvites** — `GET/POST/DELETE /tailnet/{tailnet}/user-invites/…`.
- **DeviceInvites** — `GET/POST /device/{deviceId}/device-invites/…`.
- **DevicePosture** — `GET/POST /tailnet/{tailnet}/posture/integrations/…`.
- **Services (VIP)** — `GET/PUT/DELETE /tailnet/{tailnet}/services/…`.
- **TailnetSettings** — `GET/PATCH /tailnet/{tailnet}/settings`.

Each stub group gets its own handler file (`webhooks.go`, etc.) and a corresponding
`_test.go` that simply asserts `501` and the expected JSON body — so the contract
is documented and regression-protected.

## Phase 8 — Integration Tests (Day 6–7)

### Adopt headscale's exact test infrastructure

headscale's own integration suite (`integration/`) uses
[`ory/dockertest`](https://github.com/ory/dockertest) (not `testcontainers-go`)
with a custom layered framework:

- **`hsic`** (`HeadscaleInContainer`) — runs headscale in Docker, exposes its
  gRPC + HTTP endpoints, wraps CLI calls via `docker exec`
- **`tsic`** (`TailscaleInContainer`) — runs a real `tailscaled` client in Docker
- **`Scenario`** — orchestrates pools of hsic + tsic containers and Docker networks
- Tests skip themselves unless `HEADSCALE_INTEGRATION_TEST=1` is set
  (`IntegrationSkip(t)` is the first line of every test)

We adopt **the same framework and the same skip mechanism** directly. This means:

1. Our integration tests live in `integration/` alongside the same patterns.
2. They use `ory/dockertest` (added to `go.mod`).
3. They can run locally the same way headscale's do: `go test -v -timeout 30m
   -run TestXxx ./integration/` with `HEADSCALE_INTEGRATION_TEST=1`.

### Concrete test: `TestOAuthTokenIssuance`

```go
// integration/oauth_test.go
func TestOAuthTokenIssuance(t *testing.T) {
    IntegrationSkip(t)

    pool, err := dockertest.NewPool("")
    require.NoError(t, err)
    network, err := dockertestutil.GetFirstOrCreateNetwork(pool, "headapi-test-oauth")
    require.NoError(t, err)

    // Start headscale
    hs, err := hsic.New(pool, []*dockertest.Network{network},
        hsic.WithTestName("oauth"))
    require.NoError(t, err)
    defer hs.Shutdown()
    require.NoError(t, hs.WaitForRunning())

    // Create headscale API key for headapi to use
    apiKey, err := hs.Execute([]string{"headscale", "apikeys", "create",
        "--expiration", "24h"})
    require.NoError(t, err)

    // Start headapi, pointed at headscale
    ha, err := haic.New(pool, []*dockertest.Network{network}, hs,
        haic.WithOAuthCredentials("test-client", "test-secret"),
        haic.WithHeadscaleAPIKey(strings.TrimSpace(apiKey)))
    require.NoError(t, err)
    defer ha.Shutdown()
    require.NoError(t, ha.WaitForRunning())

    endpoint := ha.GetEndpoint()

    t.Run("ValidCredentials", func(t *testing.T) {
        resp, err := http.PostForm(endpoint+"/oauth/token", url.Values{
            "grant_type":    {"client_credentials"},
            "client_id":     {"test-client"},
            "client_secret": {"test-secret"},
        })
        require.NoError(t, err)
        defer resp.Body.Close()
        assert.Equal(t, 200, resp.StatusCode)

        var tok struct {
            AccessToken string `json:"access_token"`
            TokenType   string `json:"token_type"`
            ExpiresIn   int    `json:"expires_in"`
        }
        require.NoError(t, json.NewDecoder(resp.Body).Decode(&tok))
        assert.NotEmpty(t, tok.AccessToken)
        assert.Equal(t, "Bearer", tok.TokenType)
        assert.Greater(t, tok.ExpiresIn, 0)
    })

    t.Run("WrongSecret", func(t *testing.T) {
        resp, err := http.PostForm(endpoint+"/oauth/token", url.Values{
            "grant_type":    {"client_credentials"},
            "client_id":     {"test-client"},
            "client_secret": {"wrong"},
        })
        require.NoError(t, err)
        defer resp.Body.Close()
        assert.Equal(t, 401, resp.StatusCode)
    })
}
```

### Concrete test: `TestOperatorCallSequence` (the contract test)

This is the centrepiece — it replays the exact sequence the k8s operator performs:

```go
// integration/contract_test.go
func TestOperatorCallSequence(t *testing.T) {
    IntegrationSkip(t)

    // ... setup pool, network, headscale, headapi as above ...

    // Create a user in headscale (the operator needs one to scope keys to)
    _, err = hs.CreateUser("k8s-operator")
    require.NoError(t, err)

    // Step 1: OAuth token
    token := mustGetOAuthToken(t, ha.GetEndpoint(), "test-client", "test-secret")

    authHeader := "Bearer " + token
    tailnet := "-"
    base := ha.GetEndpoint() + "/api/v2"
    client := &http.Client{Timeout: 10 * time.Second}

    // Step 2: Create auth key (what the operator does before registering a proxy)
    keyBody := `{"capabilities":{"devices":{"create":{"reusable":false,
        "ephemeral":true,"preauthorized":true,"tags":["tag:k8s"]}}},"expirySeconds":3600}`
    req := mustNewRequest(t, "POST", base+"/tailnet/"+tailnet+"/keys",
        strings.NewReader(keyBody), authHeader)
    resp := mustDo(t, client, req)
    assert.Equal(t, 200, resp.StatusCode)

    var createdKey struct {
        ID  string `json:"id"`
        Key string `json:"key"`
    }
    require.NoError(t, json.NewDecoder(resp.Body).Decode(&createdKey))
    resp.Body.Close()
    assert.NotEmpty(t, createdKey.ID)
    assert.Contains(t, createdKey.Key, "tskey-auth-")

    // Step 3: List devices
    req = mustNewRequest(t, "GET", base+"/tailnet/"+tailnet+"/devices",
        nil, authHeader)
    resp = mustDo(t, client, req)
    assert.Equal(t, 200, resp.StatusCode)
    resp.Body.Close()

    // Step 4: Delete auth key (operator cleanup)
    req = mustNewRequest(t, "DELETE",
        base+"/tailnet/"+tailnet+"/keys/"+createdKey.ID, nil, authHeader)
    resp = mustDo(t, client, req)
    assert.Equal(t, 200, resp.StatusCode)
    resp.Body.Close()

    // Verify the key no longer exists in headscale
    keys, err := hs.Execute([]string{"headscale", "preauthkeys", "list",
        "--output", "json"})
    require.NoError(t, err)
    assert.NotContains(t, keys, createdKey.ID)
}
```

## `{tailnet}` Parameter Handling

The Tailscale API uses a `{tailnet}` path parameter (e.g., `example.com` or `-`
for the default tailnet). headscale is a single-tailnet server, so headapi maps
this to a configurable `TAILNET_NAME` env var. The parameter is validated but
otherwise treated as the "owner namespace" for pre-auth key scoping.

The special value `-` (Tailscale's "default tailnet") is always accepted and
treated as equivalent to the configured `TAILNET_NAME`.

## `HeadscaleClient` Interface

The central seam for testing. All handlers receive this interface:

```go
// internal/headscale/client.go
type HeadscaleClient interface {
    // Nodes
    ListNodes(ctx context.Context, user string) ([]*v1.Node, error)
    GetNode(ctx context.Context, id uint64) (*v1.Node, error)
    DeleteNode(ctx context.Context, id uint64) error
    ExpireNode(ctx context.Context, id uint64) error
    RenameNode(ctx context.Context, id uint64, name string) (*v1.Node, error)
    SetTags(ctx context.Context, id uint64, tags []string) (*v1.Node, error)
```

## Error Handling Convention

All error responses use the Tailscale API error shape:
```json
{ "message": "human readable error" }
```

Helper:
```go
// internal/api/error.go
func writeError(w http.ResponseWriter, status int, msg string) {
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(status)
    json.NewEncoder(w).Encode(model.Error{Message: msg})
}

func notImplemented(w http.ResponseWriter, r *http.Request) {
    writeError(w, http.StatusNotImplemented, "not implemented by headapi")
}
```

gRPC status codes are mapped to HTTP status codes:
```go
func grpcStatusToHTTP(err error) int {
    switch status.Code(err) {
    case codes.NotFound:        return 404
    case codes.PermissionDenied: return 403
    case codes.Unauthenticated:  return 401
    case codes.AlreadyExists:    return 409
    case codes.InvalidArgument:  return 400
    default:                     return 500
    }
}
```

## TDD Workflow (per handler)

1. **Write the test first** in `internal/api/<group>_test.go`.
   - Use `httptest.NewRecorder()` and `httptest.NewServer()`.
   - Inject a `mockery`-generated mock for `HeadscaleClient`.
   - Assert status code, response body JSON fields, mock call count.
2. **Run tests** — all new tests fail (Red).
3. **Implement the handler** in `internal/api/<group>.go`.
4. **Run tests** — all pass (Green).
5. **Refactor** — extract translation logic to `internal/translate/`, clean up.
6. **Run tests again** — still green.

Example test structure:
```go
func TestGetDevice(t *testing.T) {
    tests := []struct {
        name       string
        deviceID   string
        mockSetup  func(*mockHeadscale.HeadscaleClient)
        wantStatus int
        wantID     string
    }{
        {
            name:     "existing device",
            deviceID: "1",
            mockSetup: func(m *mockHeadscale.HeadscaleClient) {
                m.On("GetNode", mock.Anything, uint64(1)).
                    Return(&v1.Node{Id: 1, Name: "mydevice"}, nil)
            },
            wantStatus: 200,
            wantID:     "1",
        },
        {
            name:     "not found",
            deviceID: "999",
            mockSetup: func(m *mockHeadscale.HeadscaleClient) {
                m.On("GetNode", mock.Anything, uint64(999)).
                    Return(nil, status.Error(codes.NotFound, "not found"))
            },
            wantStatus: 404,
        },
    }
    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            m := &mockHeadscale.HeadscaleClient{}
            tt.mockSetup(m)
            r := setupRouter(m)
            req := httptest.NewRequest("GET", "/api/v2/device/"+tt.deviceID, nil)
            req.Header.Set("Authorization", "Bearer "+validToken)
            w := httptest.NewRecorder()
            r.ServeHTTP(w, req)
            assert.Equal(t, tt.wantStatus, w.Code)
            if tt.wantID != "" {
                var d model.Device
                json.NewDecoder(w.Body).Decode(&d)
                assert.Equal(t, tt.wantID, d.ID)
            }
            m.AssertExpectations(t)
        })
    }
}
```

## Verification & Acceptance Criteria

The question "did we succeed?" has three levels of answer, each progressively
more convincing.

### Level 1 — Automated unit + integration tests (always runs in CI)

These are already described in each phase above. The overall bar is:

```bash
go test -race ./...                                   # unit tests, no races
HEADSCALE_INTEGRATION_TEST=1 go test -v -timeout 30m \
    ./integration/...                                 # real Docker end-to-end
```

The integration suite (Phase 8) uses exactly the same `ory/dockertest`-based
framework as headscale's own integration tests — `hsic` for the headscale
container, plus our new `haic` for headapi itself. If this suite is green we have
high mechanical confidence.

---

### Level 2 — API contract test (automated, `integration/contract_test.go`)

This is `TestOperatorCallSequence` from Phase 8 — already a real Docker test,
not a separate binary. It replays the **exact HTTP sequence the operator uses**
in order, and additionally validates every response body against the OpenAPI
schema using `github.com/getkin/kin-openapi/openapi3`:

```
1. POST /oauth/token                → 200, access_token present
2. POST /tailnet/-/keys             → 200, key with tskey-auth- prefix
3. GET  /tailnet/-/devices          → 200, devices array (may be empty)
4. DELETE /tailnet/-/keys/{id}      → 200
   └─ verify key gone from headscale CLI
```

Each response is schema-validated against the corresponding OpenAPI component.
IDs are checked for round-trip consistency (key created in step 2 is the key
deleted in step 4).

Run with:
```bash
HEADSCALE_INTEGRATION_TEST=1 go test -v -run TestOperatorCallSequence \
    ./integration/
```

---

### Level 3 — Real tailscale client registration smoke test (the ultimate acceptance test)

This uses `tsic` (`TailscaleInContainer`) — the same Docker-based real
`tailscaled` client that headscale's own suite uses — to verify that a client
can register through headscale via a pre-auth key created through headapi.

This test lives in `integration/e2e_test.go`:

```go
func TestClientRegistrationViaHeadapi(t *testing.T) {
    IntegrationSkip(t)

    // Start headscale + headapi
    hs, ha := mustStartStack(t, "e2e")

    // Step 1: get OAuth token from headapi
    token := mustGetOAuthToken(t, ha.GetEndpoint(), "test-client", "test-secret")

    // Step 2: create a pre-auth key via headapi
    key := mustCreateAuthKey(t, ha.GetEndpoint(), token, "-")
    assert.Contains(t, key.Key, "tskey-auth-")

    // Step 3: spin up a real tailscaled node and register it using that key
    tsNode, err := tsic.New(pool, "head", /* headscale version */
        tsic.WithHeadscaleName(hs.GetHostname()),
        tsic.WithCACert(hs.GetCert()),
    )
    require.NoError(t, err)
    defer tsNode.Shutdown()
    require.NoError(t, tsNode.WaitForNeedsLogin(30*time.Second))
    require.NoError(t, tsNode.Login(hs.GetEndpoint(), key.Key))
    require.NoError(t, tsNode.WaitForRunning(60*time.Second))

    // Step 4: verify node appears in headapi device list
    devices := mustListDevices(t, ha.GetEndpoint(), token, "-")
    assert.Len(t, devices, 1)
    assert.Equal(t, tsNode.Hostname(), devices[0].Hostname)

    // Step 5: delete via headapi, verify gone from headscale
    mustDeleteDevice(t, ha.GetEndpoint(), token, devices[0].ID)
    nodes, err := hs.ListNodes()
    require.NoError(t, err)
    assert.Empty(t, nodes)
}
```

**Pass criteria for Level 3:**
- [ ] Pre-auth key created through headapi is accepted by headscale
- [ ] Real `tailscaled` client successfully registers using that key
- [ ] Registered node appears in `GET /tailnet/-/devices` response
- [ ] Deleting the device via `DELETE /device/{id}` removes it from headscale
- [ ] No auth errors in headapi logs throughout

Run locally:
```bash
HEADSCALE_INTEGRATION_TEST=1 go test -v -timeout 10m \
    -run TestClientRegistrationViaHeadapi ./integration/
```

This is the **definition of done**. No Kubernetes, no `kind`, no extra tooling —
just Docker, the same way headscale tests itself.

### Observability During Verification

To debug failures during any level of testing, headapi exposes:

- **`GET /healthz`** — liveness check, should return `200 {"status":"ok"}`
- **`GET /metrics`** — Prometheus metrics including:
  - `headapi_requests_total{method,path,status}` — request counts per endpoint
  - `headapi_request_duration_seconds{method,path}` — latency histogram
  - `headapi_grpc_errors_total{method,grpc_code}` — upstream headscale errors
- **Structured logs** (slog JSON) — every request logged with `method`, `path`,
  `status`, `duration_ms`, and `error` (if any)

A failed Level 3 test with no obvious error in the operator logs should be
debugged by inspecting headapi's metrics/logs to find which API call failed and why.

### Regression Protection

Once Levels 1–3 all pass, they are added to CI as required checks on every PR.
Specifically:

1. Unit tests (`go test -race ./...`) — fast, always required.
2. Integration tests (`HEADSCALE_INTEGRATION_TEST=1 go test ./integration/...`)
   — required; run in a CI job with Docker available (same as headscale's own
   `.github/workflows/test-integration.yaml`).

This ensures:

1. No headapi change silently breaks the operator call sequence.
2. New headscale gRPC API changes are caught when the headscale module is updated.
3. The OpenAPI spec remains the source of truth — schema validation in
   `TestOperatorCallSequence` catches any drift between the spec and our responses.

## Open Questions / Future Work

- **Multi-tailnet:** headscale is single-tailnet; if multi-tailnet support is added
  upstream the translation layer would need to map `{tailnet}` to headscale users/namespaces.
- **OAuth scopes:** Tailscale's OAuth uses fine-grained scopes (`devices:read`, `auth_keys:write`, etc.).
  headapi should parse and validate scopes even if headscale doesn't enforce them,
  so that clients behave correctly.
- **Pagination:** Tailscale API documents no pagination today; if headscale grows
  very large node counts this may need server-side filtering/pagination.
- **Webhook event delivery:** Could be implemented as a headscale gRPC stream watcher
  that fans out to registered webhook URLs — a significant future phase.
- **DNS configuration:** If headscale adds DNS gRPC methods, Phase 6 stubs become real.
